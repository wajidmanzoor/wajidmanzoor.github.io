    Linear Optimization

    Theorem

    • If a Linear program has a finite optimim and there is atleast one corner point in the feasible region ,the corner point optimal exists.
    • The feasible region of a linear program(also called polyhedron) has a corner point iff it does not contian a line.i.e if its feasible region is bounded.

    Multi period inventory problem

    Consider a seller who needs to satisfy demands of a product over next T periods. For each period \(t=(1,2,3,4…T)\).demand is \(d_t\) units.The seller can order in period t at the cost of \(c_t\) per unit and there is an upper bound of \(q_t\) on the number of units that can be ordered in period \(t\). Assume that the order arrives instantaneously and any inventory that is held from period \(t\) to \(t+1\) incurs a holding cost of \(h_t\) per unit. The seller starts with zero inventory.Find the optimal ordering plan to satisfy the demand.

    Consider a 5-period inventory problem as follows.

    let \(x_i\ \forall_i \ i=1,2,..5\) be amount we produce in each period. \(I_{kl} \ \forall \ k=1,2,..5 l=1,2,..6\) be amount we carry in inventory from period k to n. here we added to variables \(I_{01}\) at begining and \(I_{5,6}\) at the end.

    Objective

    Minimize \(f(x,I)= 0I_01+22x_1 + 5I_{12}+42x_2+11I_{23}+100x_3+4I_{34}+1000x_4+0I_{45}+450x_5+0I_{5,6}\)

    Production capacity constraints

    Inventory Capacity Constraints

    Note: \(I_{01}\) must be equal to zero for \(I_{56}\) we can take any large value as it can have infinite left over.

    Non negavtive constraints

from gurobipy import*
m = Model('Five Period Inventory Problem')


production_costs = [22, 42, 100, 1000, 450]
production_capacities = [40, 100, 90, 500, 1000]

inventory_costs = [0, 5, 11, 4, 0, 0]
inventory_capacities = [50, 50, 100, 20, 1000]

demand = [10, 80, 100, 20, 440]

#create production variables
production_variables = []
for i in range(len(production_costs)):
    production_variables.append(m.addVar(vtype=GRB.CONTINUOUS, name="x{}".format(i+1), obj = production_costs[i]))

#create inventory variables
inventory_variables = []
for i in range(len(inventory_costs)):
    inventory_variables.append(m.addVar(vtype=GRB.CONTINUOUS, name="i{}{}".format(i,i+1), obj = inventory_costs[i]))

#integrate new variables    
m.update()
                    
#production capacity constraints
for i in range(len(production_costs)):
    m.addConstr(production_variables[i] <= production_capacities[i])

#inventory capacity constraints
for i in range(len(inventory_capacities)):
    m.addConstr(inventory_variables[i+1] <= inventory_capacities[i])
    
m.addConstr(inventory_variables[0] == 0)

#demand constraints
for i in range(len(production_costs)):
    m.addConstr(inventory_variables[i] + production_variables[i] - inventory_variables[i+1]  == 
                demand[i])

#optimize    
m.optimize()
print ('Model status:', m.status)

#write LP
m.write('Linear Program1.lp')

#print decision variables
for v in m.getVars():
    print (v.varName, v.x)
                
#print objective value
print ('Obj: ', m.objVal)

    Some basic terms

    Basis

    An \(m \times m\) submatrix \(B\)of \(A\) is a Basis iff \(rank(B)=m\) i.e \(B\) has \(m\) linearly independent columns of \(A\).

    Basic Variables

    Variables corresponding to columns of basis submatrix.denoted by \(x_B\).

    Non-Basic Variables

    Variables corresponding to columns of non-basis submatrix.Denoted by \(x_N\).

    Basic solution

    The solution of desicion variables corresponding to columns of Basis matrix \(x_B\)

    \(x_B=B^{-1}b\) and \(x_N=0\)

    Basic Feasible Solution It corresponds to all corner or extreme points of feasible region.

    \(x_B=B^{-1}b\) is a Basic feasible solution if \(B^{-1}b \geq 0\).

    Simplex Algorithm

    Standard form of LP is

    Maximize \(z=C^Tx\)

    Subject to \(Ax \leq b\)

    where \(x \geq 0\)

    Now standard form of linear optimatization can be written as

    Maximize \(z=C_B^Tx_B + C_N^Tx_N\)

    subject to \(Bx_B + Nx_N=b\)

    where \(x_B,x_N \geq 0\)

    Here we spilited the matrix \(A\) of order \(mxn\) into Basis submatrix \(B\) of order \(mxm\) and submatrix \(N\) of order \(mx(n-m)\).

    Let decision variables corresponding to columns of \(B\) be \(x_B\) and to that of \(N\) be \(x_N\).

    here \(x_B\) are basic vriables and \(x_n\) are non-basic variables.

    Intially we have

    If for al non basic variables \(C_N^T-C_B^TB^{-1}N \leq 0 \)(also called residue error) we have the optimal BFS at \(x_N=0\) as \(x_N \geq 0\)

    Else if for some non basic variable \(x_k\) has \(C_{N_k}^T-C_B^TB^{-1}N_k > 0\)

    So we can increase the value of \(z\) by increasing \(x_k\).

    • Entering variable \(x_k\)

    As we start increase the value of \(x_k\) some basic variables might go down to zero if that happens we have to stop increasing \(x_k\) that is called leaving variable condition.

    • Leaving variable

    \( \text{argmin} \frac{[B^{-1}b]_j}{[B^{-1}N_k]_j} :\ [B^{-1}N_k]_j>0 \)

    Here we go for the minimum of basic varaible and stop increasing \(x_k\) when that goes down to zero.

    Intial BFS

    Assuming \(b \geq 0\)

    Now

    \(Ax \leq 0\)

    Introducing slack variables we have

    \(Ax +s =0\)

    Here constraint matrix is \([A \ I]\)

    where non-basic submatrix \(A\) is \(mxn\) and basic submatrix \(I\) is identity matrix of order \(mxm\).

import numpy as np 
def tableau(A,CT,b):
    """
    Returns tableau
    parameters:
    A=Variable matrix
    CT=Transform of coefficient matrix of z
    b=b matrix"""
    nb_var=len(A[0])
    b_var=len(A)
    tab = np.zeros((b_var+1, nb_var+b_var+2))
    tab[:len(A),:len(A[0])] =np.array(A).astype("float64")
    tab[-1:,:nb_var]=np.array(CT).astype("float64")   
    tab[:-1,-1]=np.array(b).astype('float64')
    for i in range(len(tab)):
        tab[i,nb_var+i]=float(1)
    return tab

def gen_variables(tab):
    num_bvar=len(tab[:-1,0])
    num_nbvar=len(tab[0,:])-len(tab[:-1,0])-2
    b_vars=[]
    for i in range(num_bvar):
        b_vars.append("S_"+str(i))
    nb_vars=[]
    for i in range(num_nbvar):
        nb_vars.append("X_"+str(i))
    var=nb_vars+b_vars
    return b_vars,var,nb_vars

def check_residue(tab):       
    m = min(tab[-1,:])
    if m>=0:
        return False
    else:
        return True
    
def pivot_index(tab):
    n = len(tab[:,0])
    minn = min(tab[n-1,:-1])
    col = np.where(tab[n-1,:-1] == minn)[0][0]
    agmin=[]
    for i,j in zip(tab[:,col][:-1],tab[:,-1][:-1]):
        if i**2 >0:
            agmin.append(j/i)
        else:
            print("Unbounded Feasible region")
    row = agmin.index(min(agmin))
    return (row,col)

def transformations(row,col,tab):
    tab[row,:]=tab[row,:]/tab[row,col]
    for i in range(len(tab[:,0])):
        if i!=row:
            tab[i,:]=tab[i,:]-(tab[row,:]*tab[i,col])
    return tab

def change_var(b_var,var,row,col):
    b_var[row]=var[col]
    return b_var,var

def r_ans(b_var,var,tab,in_nbvar,flag):
    l_ans=[]
    print("Optimal value at: ")
    if flag:
        for i,j in zip(b_var,tab[:-1,-1]):
            l_ans.append([i,j])
            print(str(i)+"="+str(j))
    for i in in_nbvar:
        if i not in b_var:
            print(str(i)+"= 0.0")
    print("optimal Value z= "+str(tab[-1,-1]))
    return l_ans,tab[-1,-1]

def LPO(A,CT,b,tab=[],ctab=True):
    """
    returns values of variables for Maximun solution of z and maximum solution of z
    """
    if ctab==True:
        tab=tableau(A,CT,b)
    b_var,var,in_nbvar=gen_variables(tab)
    flag=0
    while(check_residue(tab)):
        flag=1
        row,col=pivot_index(tab)
        tab=transformations(row,col,tab)
        b_var,var=change_var(b_var,var,row,col)
    l_ans,optimal=r_ans(b_var,var,tab,in_nbvar,flag)
    return l_ans,optimal

    Duality Theoram

    Lets us consider two dual pair of separated locally convex spaces \((X,X)\) and \((Y,Y)\). If \(f : X \to R\) is a function then

    Primal problem can be defined as finding a value \(x\) such that \(f(x)= \sup_{x \in X}f(x)\).

    i.e if \(x\) exists \(f(x)\) is maximum of \(f\) and supremum (least upper bound). Let I be the constraints then

    Dual problem can be defined as \(f^{‘}=f+I\) it can be written as \(f^{‘}:X,Y \to R\) such that \(f^{‘}(x,0)=f(x)\)

    Duality theoram states that \(\sup_{x \in X}f(x)=\inf_{y \in Y}f^{‘}(x,y)\)

    It states that optimization problems can be viewed from either of two perspectives, the primal problem or the dual problem. The solution to the dual problem provides a lower bound or upper bound to the solution of the primal minimization problem or maximization problem respectively.The diffrence between Dual and Primal problem is called Duality Gap.
